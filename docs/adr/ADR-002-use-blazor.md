# ADR-002

## Title

Use Blazor Instead Of React

---

## Status

Accepted

---

## Context

The frontend needed a framework.

Options considered:

* Blazor
* React

React is more popular and has a larger ecosystem.

However popularity was not the primary objective.

---

## Decision

Use Blazor.

---

## Rationale

Reasons:

* Used in the workplace
* Strengthens .NET ecosystem knowledge
* Reduces context switching
* Allows focus on backend and architecture concepts

---

## Consequences

Positive:

* Faster learning of enterprise .NET stack
* Better workplace alignment
* Single ecosystem focus

Negative:

* Smaller ecosystem compared to React
* Less exposure to JavaScript frameworks

---

## Future Revisit

A React frontend may be added in the future if API-first architecture becomes a learning objective.
